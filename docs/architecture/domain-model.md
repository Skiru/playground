# Domain Model

`Place` is the publication aggregate. It has a UUIDv7 identity, unique slug,
address and geography point, one primary category, additional categories,
amenities, age zones, opening schedules, verification state, and lifecycle state.
Publication requires identity, city, coordinates, category, descriptions, and at
least one valid age zone.

`City`, `Category`, and `Amenity` are curated dictionaries. `PlaceAgeZone` stores
numeric month ranges, not marketing buckets. Weekly schedules support multiple
ordered intervals and next-day closing. Special days override weekly schedules
and can be closed or contain multiple special intervals. External references are
unique by provider and external ID but perform no import in C2.

Public reads use projections and DTOs; persistence entities are not API models.
UUIDv7 values are generated by the application through Symfony UID. Times are
represented in local schedules with an IANA place timezone; API timestamps are
ISO 8601 UTC.
